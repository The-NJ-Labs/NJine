We are building a highly specialized, terminal-native IDE for Python developers. We do **not** accept:

- Bloated GUI elements.
- Changes that break the core "Pure Python" philosophy.

## üõ† How to Contribute
1. **Open an Issue first:** Do not send a Pull Request (PR) without discussing it. We want to ensure your work fits our roadmap.
2. **Follow the Style:** We use the `Textual` framework. Ensure your UI code matches the Harlequin-inspired theme.
3. **Small PRs only:** We reject massive "re-writes." Keep your changes focused.
